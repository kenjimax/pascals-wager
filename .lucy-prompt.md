The codex `search_tool` feature has been removed and `web_search_*` are deprecated. **Use the shell instead** (you have it via `--dangerously-bypass-approvals-and-sandbox`). If you try to call a builtin web search tool and it fails or returns nothing, do not stall: switch to the shell pattern below.

To fetch a known URL:

```
curl -sL --max-time 25 -A 'Mozilla/5.0' '<url>' | sed 's/<[^>]*>//g' | tr -s ' \n' '\n ' | head -c 60000
```

(The sed strips HTML tags. Adjust `head -c` for how much you need. Use `head -300` for line-limited.)

To run a web search (no API key needed):

```
curl -sL --max-time 25 -A 'Mozilla/5.0' "https://duckduckgo.com/html/?q=$(printf '%s' '<query>' | jq -sRr @uri)" \
  | grep -oE 'https?://[^"]+' | grep -v duckduckgo | head -30
```

Pick the most relevant result URLs from the output, then `curl` each one as above.

Other reliable patterns:

- `curl -sL "https://r.jina.ai/<url>"` returns a clean LLM-friendly extract of the page (no key needed).
- For GitHub: `gh api repos/OWNER/REPO/...`, `gh search code 'foo in:file'`.
- For BigQuery / our internal data: `bq query --use_legacy_sql=false --format=pretty '<SQL>'`.
- For our shared logged-in Chrome (Google, Zendesk, etc.): `cdp open <url>` then `cdp snap -i -c`, `cdp eval '<js>'`. Per-terminal tab isolation, do not type into compose boxes.
